. Общее описание продукта
1.1 Кратко
Система УТС — это веб-интерфейс для сотрудника (агента) Ростелекома, через который он может отслеживать статус и детали одной конкретной заявки, ранее созданной через колл-центр.

Агент авторизуется, вводит номер заявки и видит:

какие услуги подключены

текущее состояние (отказ / выдан / запланирован)

при клике — полную причину отказа и мониторинг звонков

1.2 Какие проблемы решает
Агент не тратит время на звонки в колл-центр для уточнения статуса заявки

Исключается потеря информации о причинах отказа

Появляется единое окно контроля по своей заявке

Сотрудник видит только свои заявки (без доступа к чужим)

1.3 Для кого оно создано
Для агентов Ростелекома (работников, а не обычных пользователей)

Для внутреннего использования внутри компании

1.4 В каких сценариях используется
1.Агент оставил заявку через колл-центр

2.Через некоторое время заходит в УТС

3.Вводит номер заявки

Проверяет:

какие услуги уже подключены

почему отказали (если отказ)

запланирована ли работа

5.При необходимости — открывает детали для уточнения причин отказа

1.5 Общий обзор функциональности
1.Авторизация агента (роль «сотрудник»)

Поиск заявки по номеру

Отображение услуг и их статусов по одной найденной заявке

2.Детализация заявки:

причина отказа

мониторинг звонков

полная информация по заявке

Архитектура
Приложение построено по клиент-серверной архитектуре, где frontend отвечает за пользовательский интерфейс, backend — за обработку данных и бизнес-логику, а база данных — за хранение информации.

Данный функционал соответствует уровню MVP и может быть расширен в будущих версиях приложения.
