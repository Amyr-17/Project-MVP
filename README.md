# 1. Общее описание продукта

## 1.1 Кратко  
Система управления задачами — это веб-интерфейс, через который пользователь может создавать, отслеживать и управлять своими задачами  

Пользователь авторизуется и получает доступ к своим данным, где может:

- создавать задачи  
- отслеживать их статус (выполнена / в процессе)  
- редактировать или удалять задачи  
- просматривать статистику по своей активности  

Также реализована система друзей, позволяющая просматривать профили других пользователей при наличии доступа  

---

## 1.2 Какие проблемы решает  

- Упрощает управление личными задачами  
- Исключает потерю информации о задачах  
- Позволяет отслеживать прогресс выполнения  
- Обеспечивает доступ к данным только владельцу или доверенным пользователям  
- Даёт возможность централизованного контроля задач в одном интерфейсе  

---

## 1.3 Для кого оно создано  

- Для пользователей, которым важно структурировать свои задачи  
- Для личного и учебного использования  
- Может использоваться как основа для командной работы  

---

## 1.4 В каких сценариях используется  

1. Пользователь регистрируется и входит в систему  
2. Создаёт новую задачу  
3. Отслеживает её выполнение  
4. При необходимости редактирует или удаляет задачу  
5. Может добавить других пользователей в друзья  
6. Просматривает профиль и задачи (если есть доступ)  
7. Анализирует статистику в личном кабинете  

---

## 1.5 Общий обзор функциональности  

### Авторизация пользователя  
- регистрация  
- вход в систему  

### Работа с задачами  
- создание  
- редактирование  
- удаление  
- отметка выполнения  

### Система друзей  
- добавление  
- удаление  
- просмотр профилей  

### Личный кабинет  
- статистика по задачам  
- общее количество  
- выполненные и активные  

---

## Архитектура  

Приложение построено по клиент-серверной архитектуре, где:

- frontend отвечает за отображение интерфейса (HTML-шаблоны)  
- backend (Flask) обрабатывает запросы и реализует бизнес-логику  
- база данных (SQLite) хранит пользователей, задачи и связи между ними  

---

Данный функционал соответствует уровню MVP и может быть расширен в будущих версиях приложения
