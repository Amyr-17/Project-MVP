# Task Manager MVP — Requirements

## 1. Overview
Task Manager MVP — это веб-приложение для управления задачами с возможностью регистрации пользователей, создания задач и взаимодействия между пользователями через систему друзей  

Цель: предоставить удобный инструмент для управления личными задачами с возможностью расширения до командной работы  

---

## 3. Scope

### In Scope (MVP)
- Регистрация и авторизация пользователей  
- CRUD операции над задачами  
- Отображение списка задач  
- Управление статусами задач  
- Личный кабинет пользователя  
- Система друзей  
- Базовый UI интерфейс  
- Взаимодействие frontend ↔ backend через Flask  

### Out of Scope (MVP)
- сложные workflow (kanban, gantt)  
- роли и расширенные права доступа  
- уведомления  
- интеграции (email, мессенджеры и т.д.)  

---

## 4. Functional Requirements

### 4.1 User Management
- Пользователь может зарегистрироваться  
- Пользователь может войти в систему  
- Пользователь может выйти из системы  

---

### 4.2 Task Management
- Пользователь может создать задачу  
- Пользователь может редактировать задачу  
- Пользователь может удалить задачу  
- Пользователь может просматривать список задач  

---

### 4.3 Task Attributes
Каждая задача содержит:
- название (job)  
- объем работы (work_size)  
- исполнителя (team_leader)  
- список участников (collaborators)  
- статус выполнения (is_finished)  

---

### 4.4 Task Status Flow
- Задача может быть:
  - в процессе  
  - завершена  

- Пользователь может менять статус вручную  
- Отображается текущий статус задачи  

---

### 4.5 Friends System
- Пользователь может добавить другого пользователя в друзья  
- Пользователь может удалить друга  
- Пользователь может просматривать список друзей  
- Пользователь может просматривать профиль друга  

---

### 4.6 Access Control
- Пользователь видит только свои задачи  
- Доступ к профилю другого пользователя:
  - если это сам пользователь  
  - если это администратор  
  - если пользователь находится в друзьях  

---

### 4.7 Personal Dashboard
- Отображение статистики:
  - общее количество задач  
  - выполненные задачи  
  - активные задачи  
  - общее количество часов  

---

### 4.8 Backend (Flask)
- Обработка HTTP-запросов  
- Работа с базой данных  
- Управление пользователями и задачами  
- Реализация бизнес-логики  

---

### 4.9 UI (Frontend)
- Список задач  
- Формы регистрации и входа  
- Формы создания и редактирования задач  
- Личный кабинет  
- Страница пользователей и друзей  
- Dashboard со статистикой  

---

## 5. Non-Functional Requirements

### 5.1 Performance
- Время ответа сервера < 1 секунды для MVP  

### 5.2 Usability
- Простой и понятный интерфейс  
- Минимальный порог входа для пользователя  

### 5.3 Reliability
- Обработка ошибок (404, 403)  
- Валидация данных форм  

### 5.4 Security
- Авторизация пользователей  
- Ограничение доступа к данным  

### 5.5 Scalability (Future)
- Возможность добавления:
  - ролей пользователей  
  - командной работы  
  - уведомлений  
  - API интеграций  

---

## 6. Architecture (High-level)

Frontend:
- HTML шаблоны (Jinja2)

Backend:
- Flask (Python)

Database:
- SQLite

Общий паттерн:
Client → Flask → Database

## 7. Use Cases

### UC-1: Регистрация пользователя
1 Пользователь вводит данные  
2 Отправка формы  
3 Сохранение в БД  
4 Перенаправление на вход  

---

### UC-2: Создание задачи
1 Пользователь заполняет форму  
2 Отправляет данные  
3 Сохранение задачи  
4 Обновление списка задач  

---

### UC-3: Редактирование задачи
1 Пользователь открывает задачу  
2 Изменяет данные  
3 Сохраняет изменения  
4 Обновление данных  

---

### UC-4: Добавление в друзья
1 Пользователь выбирает другого пользователя  
2 Нажимает "добавить"  
3 Создаётся связь в БД  
4 Обновляется список друзей  

---

### UC-5: Просмотр профиля
1 Пользователь открывает профиль  
2 Проверяется доступ  
3 Отображаются задачи пользователя  

---
